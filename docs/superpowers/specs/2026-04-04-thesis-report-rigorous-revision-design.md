修订范围

1. 代码层
   - 检查并修正 `src/mcmc_optimized.py` 与 `src/mcmc_option_pricing.py` 中 MTM 实现细节：
     - 权重计算
     - 辅助变量生成
     - 接受概率公式实现一致性
   - 对 RWMH 变量命名/表达做必要规范化（如 `log_ratio`），确保正文公式与实现一致。

2. 实验层
   - 运行 `src/mcmc_optimized.py` 获取主实验指标：价格、误差、IAT、ESS、运行时间、接受率。
   - 以不同随机种子（42~46）做 5 次独立重复实验，补充“均值±标准差”。
   - 运行 `src/mcmc_advanced.py` 获取收敛证据（Geweke，必要时补充 R-hat 或多链轨迹说明）。
   - 运行 `src/visualization_optimized.py` 生成最新图并用于论文引用。

3. 论文文本层
   - 更新第4章表4-1/4-2/4-3为最新运行结果。
   - 在4.2补充“调试后 MTM 价格估计收敛至解析解附近”的基于实测的表述。
   - 在4.6补充“5次独立重复实验结果一致性”陈述。
   - 在5.2与4.7强化“统计效率提升与时间成本权衡、串行实现成本高”的总结。
   - 在2.3补写目标分布数学形式（对 \(\ln S_T\) 的正态密度）。
   - 统一图表编号与引用格式，删除重复/无效图引用。
   - 保持用户先前约束：全文不出现 `#` 字符。

执行约束

- 所有结论以“本次运行结果”为准，不再沿用与当前代码不一致的历史数值。
- 每张表的数值必须可追溯到具体脚本与参数。
- 若某项运行失败，需记录失败原因并采用可解释备选方案（但优先保证真实运行结果）。

复现实验固定配置（写入附录）

- Python 版本：运行环境检测后写入（如 `python --version` 输出）。
- 核心依赖版本：`numpy`, `scipy`, `matplotlib`（通过脚本打印版本并记录）。
- 随机数源：`numpy.random.seed`。
- 主比较配置：`n_samples=20000`, `burn_in=n_samples//4`, `proposal_std=0.3`。
- 基准比较配置：`n_samples=50000`, `burn_in=n_samples//4`。
- 多链高级配置：按 `mcmc_advanced.py` 默认参数并记录。

统计验收阈值

- Geweke：\(|z| < 1.96\) 视为通过（5% 双侧）。
- 若计算 R-hat：\(\hat{R} < 1.01\) 视为通过。
- ESS/IAT 不设绝对阈值，但需保证“MTM-K4 相比 RWMH 在 IAT 下降方向一致”。

五次重复汇总口径

- 以“算法 × 指标”为单元，对 5 次实验计算均值与标准差。
- 在表4-1新增“价格均值±std”列。
- 在4.6中给出“方向一致性”结论，不做过度显著性外推。

追溯矩阵（交付时补全）

- 章节/段落 -> 表/图编号 -> 数据来源脚本 -> 参数 -> 随机种子 -> 运行时间戳。
- 至少覆盖：2.3、4.2、4.3、4.5、4.6、5.2。

失败降级顺序与停止条件

1. 必保：`mcmc_optimized.py` 主实验 + 表4-1/4-2/4-3更新。
2. 次保：5次独立重复实验与均值±std列。
3. 可降级：`mcmc_advanced.py` 的 R-hat（若不可得则仅保 Geweke）。
4. 若主实验无法跑通：停止并向用户报告，不进行“虚构填表”。
