# 本科毕业论文研究报告（Markdown）设计说明

## 1. 目标

在 `master` 分支现有代码基础上，生成一份符合数学与统计学院本科毕业论文风格的研究报告 Markdown 文档，主题聚焦“多提案 MCMC 在 Black-Scholes 期权定价中的效率研究”，并保证在 WPS 导出 PDF 后约 12–16 页。

目标文件：`本科毕业论文-研究报告.md`

## 2. 约束与定位

- 学术风格：本科毕业论文规范写法（非教程、非纯项目说明）。
- 深度配比：理论与实验平衡（约 5:5）。
- 与仓库一致：内容必须映射 `master` 分支已有代码与实验结构，不捏造不存在模块。
- 可答辩：每章尽量具备“可口述”的逻辑链（研究问题 → 方法 → 结果 → 结论）。

## 3. 章节结构（拟定）

1. 题目、摘要、关键词（中英文）
2. 第 1 章 绪论
   - 研究背景与意义
   - 国内外研究现状
   - 研究问题与本文贡献
3. 第 2 章 理论基础
   - Black-Scholes 模型与解析定价
   - Monte Carlo 与 MCMC 基础
   - RWMH 与 MTM 原理
   - 诊断指标（接受率、ACF、IAT、ESS）
4. 第 3 章 方法与实现
   - 项目架构与代码路径说明
   - 核心算法流程与伪代码
   - 实验流程（参数设置、采样步骤、评价方法）
5. 第 4 章 实验结果与分析
   - 结果表格整理
   - 时间效率与统计效率权衡
   - 与理论预期的一致性及偏差解释
6. 第 5 章 结论与展望
7. 参考文献（占位，GB/T 7714 风格条目）
8. 附录（运行命令、关键函数说明）

## 4. 内容来源与映射

主要参考仓库文件：

- `README.md`
- `README2.md`（仅用于概念核对，不直接复用口语化表述）
- `src/mcmc_option_pricing.py`
- `src/mcmc_optimized.py`
- `src/mcmc_advanced.py`
- `src/visualization_optimized.py`

报告会明确标注这些模块在研究流程中的作用，确保“论文化叙述”与“代码事实”一致。

事实分级与冲突处理规则：

- 一级事实：`src/*.py` 的可复现实验逻辑与可运行输出。
- 二级事实：`README.md`、`README2.md` 的历史记录数据与解释。
- 若一级与二级事实冲突，以一级事实为准；正文中说明差异来源，不隐去冲突。
- 结果首次出现时给出来源标注：`(source: 路径, 参数配置, 随机种子)`。

## 5. 篇幅控制策略（WPS PDF 12–16 页）

- 摘要与关键词：约 1 页
- 第 1 章：1.5–2 页
- 第 2 章：3–4 页
- 第 3 章：2–3 页
- 第 4 章：3–4 页
- 第 5 章 + 参考文献 + 附录：2–3 页

版式控制约束（用于稳定分页）：

- 使用三级以内标题（`#`/`##`/`###`），避免过深层级。
- 公式统一采用 LaTeX 语法，行间公式与行内公式混排。
- 每章至少 1 张表或图（可复用仓库图，需注明来源）。
- 单段建议不超过 250 字，避免过长段导致页面拥挤。

## 6. 验收标准（Definition of Done）

必须同时满足以下条件：

- 形成 1 个完整文件：`本科毕业论文-研究报告.md`。
- 章节完整：摘要（中英文）、第1章至第5章、参考文献、附录全部具备。
- 内容量满足十几页目标：WPS 导出 PDF 预计不少于 10 页，目标 12–16 页。
- 结构要素不少于：3 个表格、3 个图示引用位、8 条及以上公式表达。
- 参考文献不少于 8 条，采用 GB/T 7714 风格书写（非“占位符”）。
- 每个核心实验结论可追溯至仓库文件与参数配置。
- 全文为学术书面语，不出现教程式口语表达。

## 7. 风险与处理

- 风险：已有 README 中部分实验数字与实现版本可能不完全同步。
  - 处理：执行事实分级规则；冲突时以 `src` 可复现逻辑为准，并在正文说明差异。
- 风险：Markdown 到 WPS/PDF 的分页受字体和行距影响。
  - 处理：通过分级标题、段落长度和表格密度控制在十几页范围。
- 风险：后续复现环境差异导致结果轻微偏移。
  - 处理：在附录中固定分支、给出关键命令、参数与随机种子；标注“本报告结果对应 master 分支”。

## 8. 交付物

- 新增文件：`本科毕业论文-研究报告.md`
- 内容完整可直接用于后续粘贴到学校论文模板中进行格式化。
